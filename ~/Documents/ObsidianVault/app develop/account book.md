# 代购记账 App — 业务逻辑文档

> 版本：v2.0.0 | 更新日期：2026-03-31

## 核心业务流程

```
客户下单 → 创建预购订单 → 标记"买到"(库存+) → 收款发货 → 已付+已发 → 自动归档 → 统计报表
```

---

## 主线流程详解

### 1. 预购阶段

用户填写：分类、客户、商品、数量、进价、售价、定金、备注

- 分类/客户/商品不存在时自动创建
- 选择分类后，商品下拉自动过滤为该分类下的商品
- 选择已有商品自动填入进价和售价
- 支持三种添加模式：
  - 单次添加
  - 连续换买家（锁定商品信息，循环填客户）
  - 连续换商品（锁定客户，循环填商品）
- 创建预购的同时自动创建一条 fulfillment 记录

预购列表支持两种视图：
- 按商品分类 → 分类下按商品聚合，显示总件数，可折叠
- 按买家分组 → 可折叠

### 2. 采购阶段（"买到"按钮）

- 二次确认后标记 `isBought = true`
- 商品总库存 += 数量
- 该订单从预购列表消失，流转到收款发货

### 3. 收款发货阶段

按客户分组，每条订单有两个独立状态：

- 切换"已付" → 更新 isPaid
- 切换"已发" → 更新 isShipped，同时调整批次库存（已发 -数量，取消 +数量）
- 筛选：全部 / 未完成 / 已完成

**已付 + 已发 都完成时自动归档：**
- 商品总库存 -= 数量
- 写入 purchaseHistory（快照客户、商品、价格、时间）
- 删除 preorder 和 fulfillment 记录

### 4. 账单生成

- 按客户生成账单，可编辑数量、进价、售价、定金、付款状态
- 可设置邮费
- 三个汇总：总售价（自动算）、已付（手动填）、待付（= 总售价 - 已付）
- 生成账单预览卡片和报告页面

### 5. 统计模块

- 从 purchaseHistory 按日期范围查询
- 支持按买家筛选
- 汇总总成本、总优惠、总收入、总利润
- 点击任意记录行弹出编辑弹窗，可一次性修改买家、商品、数量、进价、收入、完成日期
- 编辑后的数据仅保存在 purchaseHistory 中，不回写到商品库、客户表等其他数据
- 收入字段保存时自动反算单价（收入 ÷ 数量 = salePrice）

## 辅助模块

- **商品库**：CRUD + 批次管理（按日期分批，独立库存和价格）
- **客户管理**：CRUD + 购买历史展示
- **分类管理**：CRUD + 分类下商品列表

## 数据保护机制

```
数据变更 → 保存到 IndexedDB + localStorage 快照
                                    ↓
每天首次打开 → 自动上传到 Cloudflare KV（需设置同步密钥）
                                    ↓
数据丢失时 → 先查 localStorage → 再从 Cookie 找回密钥 → 自动从云端恢复
```

- `navigator.storage.persist()` 请求持久化存储
- 同步密钥同时存 localStorage + Cookie（1年有效）
- 手动导出/导入 JSON 备份文件

## 库存变动规则

| 操作 | 商品总库存 | 批次库存 |
|------|-----------|---------|
| 标记"买到" | +数量 | — |
| 标记"已发" | — | -数量 |
| 取消"已发" | — | +数量 |
| 已付+已发归档 | -数量 | — |
| 归档回退 | +数量 | — |
| 手动加批次 | — | +增量 |

## 数据模型（7个 IndexedDB Store）

```
categories ←── products ←── preorders ──→ fulfillments
                  ↑              ↓               ↓
            productBatches    customers     purchaseHistory
                                ↑               ↑
                                └───────────────┘
```

- **categories** — 商品分类
- **customers** — 客户
- **products** — 商品（含库存、进价、售价）
- **preorders** — 预购订单（关联分类、客户、商品，含定金）
- **fulfillments** — 履约记录（付款、发货状态）
- **productBatches** — 商品批次（按日期分批，独立库存和价格）
- **purchaseHistory** — 已完成交易的归档记录

preorder 是核心枢纽，关联分类、客户、商品。fulfillment 跟踪履约状态。归档后数据快照到 purchaseHistory，原始记录删除。

## 技术架构

- 纯前端 PWA，无后端服务器
- 原生 JS，零依赖，无框架
- IndexedDB 存储数据
- Service Worker 离线缓存（cache-first）
- Cloudflare Workers KV 云端备份
- GitHub Pages 托管
